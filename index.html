13. aria-* attributes

Purpose: Provide extra semantic information about UI elements to assistive technologies.They do not affect visual display but help users who rely on accessibility tools understand and interact with your site better.Used especially for custom widgets or interactive elements that don’t have built-in semantic meaning.

1. aria-label Provide accessible label Icon-only buttons

What it does: Provides a text label for an element that may not have visible text.Use case: When a button or icon has no visible label but needs to be described.Screen readers will read "Close menu" when this button is focused, even though the button only shows an icon.

2. aria-labelledby Reference visible label Sections and regions
Your username must be 6-12 characters long.

What it does: References the ID of another element that labels this one.Use case: When a visible label exists elsewhere on the page.Screen readers associate the heading "User Settings" with this region, giving context.

3. aria-hidden Hide element from screen readers Decorative icons

What it does: Marks elements as hidden from assistive technologies.Use case: Hiding decorative elements or content not relevant to screen readers.The star icon is ignored by screen readers, reducing clutter.

4. aria-pressed Toggle button state Bold/italic toggle buttons

What it does: Indicates a toggle button's pressed state (true or false).Use case: Custom toggle buttons or switches.Screen readers announce whether the button is pressed or not.
