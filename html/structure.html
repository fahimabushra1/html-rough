why use semantic?

  • Improves SEO (Search engines understand the content better).
  • Helps accessibility (Screen readers can interpret content properly)
  • Makes code easier to read & maintain

list of semantic tag

  • Structural Tags
    • header → Defines the header of a page or section.
    • nav → Represents navigation links.
    • section → Groups related content within a page.
    • article → Represents an independent piece of content (e.g., a blog post).
    • aside → Represents content related to the main content (e.g., sidebars).
    • footer → Defines the footer of a page or section.
    • main → Represents the main content of the document.
  • Text & Content Tags
    • h1-h6 → Headings that define content hierarchy.
    • p → Defines a paragraph.
    • mark → Highlights text for emphasis.
    • time → Represents dates or times.
    • address → Defines contact information.
    • blockquote → Represents a quoted section from another source.
    • cite → Specifies a reference or citation.
    • abbr → Defines an abbreviation (e.g., HTML for HyperText Markup Language).
    • code → Represents a piece of computer code.
